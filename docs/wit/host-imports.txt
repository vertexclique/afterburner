# Authoritative manifest of every host import declared at the
# `afterburner:host` namespace. One name per line, alphabetical.
# Lines starting with `#` and blank lines are ignored.
#
# This file is the single source of truth for what host functions
# exist. The `abi_parity` test in afterburner-wasi/tests asserts
# that THIS manifest, the plugin's `extern "C"` block in
# `crates/afterburner-plugin/src/host_api.rs`, and the wasi linker
# registrations in `crates/afterburner-wasi/src/host_imports.rs`
# all list the same names. Adding a host import is now a 4-place
# edit (manifest + extern + linker + JS bridge); removing one is
# the same in reverse. The test fails CI on any drift.
#
# The semantic-level WIT spec at docs/wit/afterburner-host.wit
# remains the typed contract — update it alongside this manifest
# when adding new imports so reviewers can see the shape.
