Run real Node apps. On your terms. At full burn.

Afterburner is a JavaScript runtime built in Rust. burn server.js stands up Express, Postgres clients, Redis pubsub. burn npm install works. TypeScript works. Worker threads, raw TCP, TLS with SNI, WebAssembly modules, native-addon shadows for bcrypt, sqlite3, sharp: all in. Every capability (files, network, env, processes) is yours to grant or deny, per script.

01

Real Node apps, unmodified

Express servers, Postgres clients, Redis pubsub. burn server.js binds ports, opens raw TCP / TLS, drives the event loop, handles SIGINT cleanly.

02

Capability-gated everything

Files, network, environment, subprocesses. Every host capability is gated. Pass --allow-fs=./out and that's all the script gets.

03

TypeScript, no toolchain

burn app.ts just runs. .mts, .cts, ESM, CommonJS. Types stripped, modules translated, no toolchain.

04

Limits you actually set

Fuel, memory, wall-clock. Every dimension is yours to pick. A runaway script loses its return value, not your host.

05

Workers + scheduler

worker_threads ships. One flag turns a single instance into an N-worker pool. Bounded queues, work-stealing, graceful drain. Capability inheritance never widens.

06

Pass-through for npm & friends

burn npm install, burn pnpm run dev, burn npx tsc. Every nested node call routes back through the runtime.

Every limit, every gate,
configurable.

      
Nothing is hardcoded. Set the budgets that match your workload. Default-deny capabilities; grant exactly the surface a script needs. Every knob is exposed on the CLI and the embedded API.

    


    

      

        

          Fuel
          configurable
        

        
Operations budget

        
A deterministic op counter. Tight loops trip the gauge regardless of host speed. Set it to whatever bounds your workload, or disable it entirely.

        
.fuel(your_budget)

        

      


      

        

          Memory
          configurable
        

        
Per-script memory cap

        
Hard cap enforced by the sandbox allocator. Over-budget allocations surface as JS-side OutOfMemory. Your host never gets OOM-killed.

        
.memory_bytes(your_cap)

        

      


      

        

          Timeout
          configurable
        

        
Wall-clock kill switch

        
Backstop for I/O that doesn't burn fuel. Independent of fuel and memory. Set it in milliseconds, seconds, minutes, whatever fits.

        
.timeout_ms(your_deadline)

        

      


      

        

          Workers
          configurable
        

        
Scheduler width

        
From single-threaded to as many workers as your machine has cores. Bounded queues and work-stealing keep cores busy without thrashing locality.

        
.threaded(N)

        

      


      

        

          Filesystem
          scoped
        

        
Read & write roots

        
Whitelist exact paths. FsAccess::None, ReadOnly(roots), or ReadWrite(roots). Nothing you don't grant.

        
--allow-fs=./data,/tmp/jobs

      


      

        

          Network
          scoped
        

        
Hostname allow-list

        
Outbound HTTP and DNS gated by hostname. Wildcards supported. The script can reach exactly the services on your list, and nothing else.

        
--allow-net=api.example.com,*.trusted.io

Multi-tenant extensions

        
Let customers ship JavaScript that augments your product. Capability gates pin tenant N's fs reads to /tmp/tenant-N. Limits keep their bugs out of your CPU graphs.

        
customer scriptingisolation

      

      

        

          
        

        
Per-row UDFs in your data plane

        
Ship a streaming transform as a JS file. Pipe rows in via burn thrust transform.js; let the scheduler hash-route by key and steal under skew.

        
streamingtransforms

      

      

        

          
        

        
Locked-down edge handlers

        
Each request gets a sealed manifold by default. Outbound HTTP needs an allow-list; fs stays off until you grant it. Production-shaped isolation, locally testable.

        
edge computehandlers

      

      

        

          
        

        
Scriptable build steps

        
Replace ad-hoc shell with version-pinned JS that has only the capabilities the step needs. burn --allow-fs=./out --allow-env=CI build.ts beats a 200-line Bash file.
