Same commands you already know. burn server.js, burn npm install, burn app.ts. They run through the Rust runtime. Real ports, real sockets, TypeScript stripped on the way in. Adaptive by default: native ignition on the first call, fully sandboxed Wasmtime from the second.

Built to drop into your existing JavaScript workflow, and then give you the controls Node never offered.

01

Real Node apps, unmodified

Express servers, Postgres clients, Redis pubsub, MongoDB drivers. burn server.js binds ports, opens raw TCP, terminates TLS, drives the event loop, handles SIGINT cleanly.

02

Capability gates via Manifold

fs, net, env, crypto, child_process: each is a field on the Manifold. Library default is sealed(). Grant only the surface a script needs; require for anything else throws clearly.

03

TypeScript & ESM, no toolchain

burn app.ts just runs. .ts, .mts, .cts stripped via oxc; ESM import/export transformed to CommonJS at load. No tsc, no bundler.

04

Adaptive ignition (Flying Start)

Native QuickJS on the first call for sub-millisecond startup; Wasmtime + Javy plugin from the second. Same script ID across both. Pin a single mode if you'd rather.

05

Limits you actually set

.fuel(N), .memory_bytes(N), .timeout_ms(N): three independent caps, all enforceable per call. Wall-clock backstops I/O that doesn't burn fuel. Runaway scripts lose their return value, not your host.

06

Workers + lock-free scheduler

worker_threads ships (process-per-worker, length-prefixed JSON IPC). .threaded(N) turns one instance into a steal-when-idle pool with token-bucket admission. Capability inheritance never widens.

07

Native-addon shadows

require('bcrypt'), argon2, jsonwebtoken, sqlite3, sharp all resolve inside the WASM sandbox to pure-Rust hosts (rusqlite + bundled SQLite C; image + SIMD fast_image_resize). Same npm API, no .node binary.

08

WebAssembly modules in JS

WebAssembly.compile / instantiate / Module / Instance / Memory backed by a per-host wasmtime sub-runner. SQL.js, @jsquash, libheif-js, anything that compiles to wasm32-wasi with no env imports.

09

Embed in Rust with one crate

cargo add afterburner. Afterburner::builder() → .register(src) → .run(id, json). Pluggable BurnCacheBackend (Redis / S3 / SQLite); pluggable StateStore; HostContext hooks for log / emit_row / get_env.

Every limit, every gate,
configurable.

      
Nothing is hardcoded. Set the budgets that match your workload. Default-deny capabilities; grant exactly the surface a script needs. Every knob is exposed on the CLI and the embedded API.

    


    

      

        

          Fuel
          configurable
        

        
Operations budget

        
A deterministic op counter. Tight loops trip the gauge regardless of host speed. Set it to whatever bounds your workload, or disable it entirely.

        
.fuel(your_budget)

        

      


      

        

          Memory
          configurable
        

        
Per-script memory cap

        
Hard cap enforced by the sandbox allocator. Over-budget allocations surface as JS-side OutOfMemory. Your host never gets OOM-killed.

        
.memory_bytes(your_cap)

        

      


      

        

          Timeout
          configurable
        

        
Wall-clock kill switch

        
Backstop for I/O that doesn't burn fuel. Independent of fuel and memory. Set it in milliseconds, seconds, minutes, whatever fits.

        
.timeout_ms(your_deadline)

        

      


      

        

          Workers
          configurable
        

        
Scheduler width

        
From single-threaded to as many workers as your machine has cores. Bounded queues and work-stealing keep cores busy without thrashing locality.

        
.threaded(N)

        

      


      

        

          Filesystem
          scoped
        

        
Read & write roots

        
Whitelist exact paths. FsAccess::None, ReadOnly(roots), or ReadWrite(roots). Nothing you don't grant.

        
--allow-fs=./data,/tmp/jobs

      


      

        

          Network
          scoped
        

        
Hostname allow-list

        
Outbound HTTP and DNS gated by hostname. Wildcards supported. The script can reach exactly the services on your list, and nothing else.

        
--allow-net=api.example.com,*.trusted.io

node_modules

✓

        

          
Package managers

          
burn bun add installs end-to-end — bun is a native binary, args pass through cleanly. burn npm / pnpm / yarn are best-effort: the PATH shim re-enters the sandbox so every nested node routes home, but their async dispatch chains exercise more Node fidelity than the runtime currently provides. Use bun if you want it to just work today.

Multi-tenant extensions

        
Let customers ship JavaScript that augments your product. Capability gates pin tenant N's fs reads to /tmp/tenant-N. Limits keep their bugs out of your CPU graphs.

Locked-down edge handlers

        
Each request gets a sealed manifold by default. Outbound HTTP needs an allow-list; fs stays off until you grant it. Production-shaped isolation, locally testable.
